一、git对象模型
SHA
所有用来表示项目历史信息的文件，是通过一个40个字符的“对象名”来索引的。
6ff87c4664981e4397625791c8ea3bbb5f2279a3
每一个“对象名”都是对“对象”内容做SHA1哈希计算得来的。
两个不同内容的对象不可能有相同的“对象名”

对象：包括三个部分 类型、大小、内容
对象的类型：blob tree commit tag
blob 一个blob对象就是一块二进制数据，他没有指向任何东西或有任何其他属性，甚至连文件名都没有。
tree 一个tree对象有一串指向blob对象或是其他tree对象的指针，它一般用来表示内容之间的目录层次关系。
commit commit对象指向一个tree对象，并且带有相关的描述信息。
tag
一个标签对象包括一个对象名(译者注:就是SHA1签名), 对象类型, 标签名, 标签创建人的名字("tagger"), 还有一条可
能包含有签名(signature)的消息. 

与SVN的区别：SVN存储每次提交之间的差异。git会把每次提交的文件的全部内容都会记录下来。
